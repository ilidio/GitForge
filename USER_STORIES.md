# User Stories - GitForge MVP

## 1. Repository Management
- **US1.1: Open Local Repo**
  As a user, I want to select a local folder so that I can manage an existing Git repository.
- **US1.2: Clone Repo**
  As a user, I want to provide a URL and a destination folder so that I can clone a remote repository.
- **US1.3: Repo List**
  As a user, I want to see a list of my recently opened repositories for quick access.

## 2. Working Directory & Staging
- **US2.1: View Status**
  As a user, I want to see a list of modified, staged, and untracked files so that I know the state of my project.
- **US2.2: Stage/Unstage**
  As a user, I want to click a button to stage or unstage specific files.
- **US2.3: View Diff**
  As a user, I want to see the changes made to a file before staging/committing.

## 3. Committing
- **US3.1: Create Commit**
  As a user, I want to enter a commit message and commit my staged changes.
- **US3.2: Amend Commit**
  As a user, I want to amend my last commit to fix minor mistakes.

## 4. History & Branching
- **US4.1: Branch Graph**
  As a user, I want to see a visual graph of commits and branches to understand the project history.
- **US4.2: Checkout Branch**
  As a user, I want to switch between branches by selecting them from a list or graph.
- **US4.3: Create Branch**
  As a user, I want to create a new branch from my current HEAD.

## 5. Remote Operations
- **US5.1: Pull/Push**
  As a user, I want to pull changes from and push changes to the remote repository.
- **US5.2: Fetch**
  As a user, I want to fetch updates from the remote without merging them immediately.
